Elfentier SNN Current Architecture

2026-06-23時点で実装済みのSNN学習環境の構造図。Chrome拡張がブラウザ行動とメディア刺激を観測し、SNNモデルを逐次更新し、`.edensnn` として保存、分析、会話利用できる。

全体フロー

Browser World ページDOM、可視テキスト、入力、クリック、スクロール、選択、video/audio再生状態
Content Script 刺激をイベント化。入力値は設定により metadata / text / full を選択
Background Worker 設定を読み、CPU/WebGPU補助を選択してSNNへイベントを渡す
SNN Core 語彙ニューロン、シナプス、報酬、遅延報酬、神経修飾、固定化を更新
Storage / Export chrome.storage.localに永続化し、`.edensnn` モデルとして出力
Dashboard / Chat EDEN dashboardで分析。snn-chat-labで学習済み記憶と会話

学習ステップ

1. Tokenize

イベント種別、ページテキスト、メディア状態、モラル抽象トークンを結合し、最大480トークンを処理する。

2. Reward & Salience

操作種別、新奇性、メディア継続、モラル設定から即時報酬と顕著性を計算する。

3. Spike Update

語彙ニューロンの膜電位を更新し、閾値を超えた場合に正/負スパイクを発火する。

4. STDP-like Weight Update

隣接トークンのシナプス重みを、報酬、神経修飾、安定度、設定に基づいて更新する。

5. Delayed Reward

最近活動したシナプスをeligibility traceに残し、後のクリック/選択/動画完走で過去へクレジットを戻す。

6. Stabilize

恒常性可塑性で閾値を調整し、一定ステップごとに高顕著性記憶を睡眠リプレイ風に固定化する。

設定と性能

Learning Settings

  • Input capture: metadata / text / full
  • Sensitive text: redact / abstract / full
  • Moral learning: off / observe / shape / constraint
  • Max vocabulary: 0なら無制限
  • Compute backend: auto / cpu / webgpu

Performance Metrics

  • Model size: JSON化したモデル概算容量
  • CPU load: 学習ステップ時間からの推定負荷
  • Step time / Avg step: 処理時間
  • Backend: CPUまたはWebGPU
  • WebGPU: 利用可否
