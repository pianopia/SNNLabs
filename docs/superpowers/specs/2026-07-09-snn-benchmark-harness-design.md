# SNN Benchmark Evaluation Harness + 3DCG Construction Benchmark — Design

作成日: 2026-07-09
状態: 承認済み(brainstorming フェーズ完了、実装計画へ移行)

## 背景と目的

このリポジトリ (SNNLabs / Elfentier) は、3つのランタイム
(PyTorch DST-SNN、EDEN/ブラウザ TypeScript 身体SNN、Nim 参照実装) に
またがる SNN 実験基盤の集合体である。最終目標は
「LLMより高精度・能動的・自律的で、空間把握・瞬発性・身体操作性を持つ、
低消費電力・低リソースの AI モデル」の開発。

**戦略上の核心**: LLM と言語・汎用知識で戦わない。SNN が構造的に勝てる
「閉ループ・低レイテンシ・低電力の反応/空間/身体タスク」に土俵を絞り、
そこで「精度」を **具体的タスクの成功率** として再定義し、
LLM 系方策を **品質・レイテンシ・エネルギー** の3軸で比較する。

現状の最大の障害は **タスク定義・評価指標・ベースラインが存在しない** こと。
本設計はこの Phase 0(計測基盤)を確立する。

## スコープ

### 含む (Phase 0)
1. リポジトリ役割の統合(学習正典 = PyTorch DST-SNN)
2. 共通評価ハーネス(全ベンチマーク横断の指標 + 対LLMベースライン)
3. ニューロモルフィックベンチマーク: N-MNIST, DVS128 Gesture
4. EDEN14 image→3D 構築ベンチマークのスコアラ(SketchFab 完成品比較)
5. 既存 Web 学習器の2バグ修正(Phase 1 頭金)

### 含まない (後続フェーズ)
- 3DCG 生成器の本実装(Track 1 Blender エージェント / Track 2 直接ジオメトリ回帰)
  — 本設計ではスコアラと自明なベースライン生成器までを作り、生成器本体は別スペック。
- 空間エンコーダ(place/grid セル)、World-Model、LLM ルーター(Phase 2-4)

## ベンチマークスイート(3系統)

| 系統 | タスク | 主指標 | 主張 |
|---|---|---|---|
| A. N-MNIST | イベントカメラ MNIST 分類 | 分類精度(%) | 同等/低エネルギーでの精度 |
| B. DVS Gesture | DVS128 11ジェスチャ分類 | 精度 + 判断確定レイテンシ | 瞬発性 |
| C. EDEN14 image→3D | 参照画像から3Dアセット構築 | 3DCG スコアラ(下記) | 空間把握・身体操作性・構築能力 |

## コンポーネント設計

### 1. 共通評価ハーネス `src/dst_snn/eval/`

責務: どのベンチマークも同一スキーマの結果を出す。UI/レポートは結果スキーマにのみ依存。

- `metrics.py`: 指標計算ユニット。各関数は純粋関数として単体テスト可能。
  - `quality`: 精度 or タスク固有スコア
  - `latency`: 推論あたり ms、p50/p95
  - `sparsity`: 推論あたりスパイク数、活性ニューロン率
  - `energy`: SNN の AC(加算)演算数 vs dense baseline の MAC 演算数を、
    設定可能なエネルギーモデル(既定 45nm: 0.9 pJ/MAC, 0.1 pJ/AC)で換算。
    可能なら macOS `powermetrics` で壁時計エネルギー実測を併記。
  - `model_size`: パラメータ数・バイト数
- `runner.py`: ベンチマークが実装する `BenchmarkRunner` プロトコル
  (`prepare()`, `run() -> RunResult`)と共通実行ループ。
- `result.py`: JSON 結果スキーマ(`RunResult`, `MetricSet`)とレポート出力。
- `baselines/`: 対LLM/対ANN ベースライン方策の共通インターフェース。
  A/B はラスタライズフレームを LLM に与える方式、C は LLM に同じ Blender 操作を
  駆動させる方式。

エネルギーモデル定数は設定ファイルで上書き可能とし、出典を結果メタに記録する。

### 2. ニューロモルフィックランナー `benchmarks/neuromorphic/`

- `datasets.py`: N-MNIST, DVS128 Gesture のローダ。イベント列 → 時間ビン化された
  スパイクテンソル `[batch, time, features]`(DendriticSNN の入力形状に整合)。
- `run_nmnist.py`, `run_dvs_gesture.py`: 既存 `DendriticSNN` を分類ヘッド付きで学習/評価。
  必要に応じ `ChronoPlasticLIFLayer` を前段に接続(時間文脈の付与)。
- DVS Gesture は「判断確定レイテンシ」= イベント列の何割時点で正解クラスが
  安定確定するかを計測(瞬発性の指標)。
- ベースライン: 小型 ANN/CNN、および(可能なら)ラスタライズフレームを与えた LLM。

### 3. 3DCG スコアラ `benchmarks/threedcg/`

責務: **任意の生成アセット** を SketchFab 参照と比較して採点する。生成器から独立。
実装は headless Blender (bpy) + trimesh。各指標は独立関数で、参照アセットに
該当データが無い場合(例: 参照が非リグド)は当該指標を `null` として除外。

- **ジオメトリ** `geometry.py`: Chamfer 距離、体積 IoU(正規化整列後)、法線整合性
- **トポロジ** `topology.py`: ポリゴン/頂点数比、多様体・水密チェック、n-gon 率
- **UV 展開** `uv.py`: チャート数、UV 被覆率、ストレッチ/歪み(三角形ごとの面積・角度歪み)、重なり率
- **リギング/ボーン** `rig.py`: ボーン数一致、スケルトン階層の編集距離、対称性、関節位置誤差
- **スキニング/ウェイト** `skin.py`: ウェイト正規化(合計=1)、頂点あたり最大影響数、
  ウェイト場の平滑性(Laplacian)、孤立ウェイト検出
- **テクスチャリング** `texture.py`: 解像度、PBR チャネル完全性
  (albedo/normal/roughness/metallic)、一致視点レンダの LPIPS/SSIM
- **横断**: 構築レイテンシ + 対LLM 3Dエージェントのエネルギー比(共通ハーネス経由)

入力: 参照画像(SketchFab モデルのレンダ)。参照コーパス仕様は `corpus.md` に定義。
検証用に自明なベースライン生成器(凸包 / プリミティブフィット)を同梱し、
スコアラを end-to-end で検証する。

### 4. 生成アーキテクチャ(長期ロードマップ、本スペックでは非実装)

両トラックは参照画像スパイキングエンコーダ・ハーネス・参照コーパスを共有:
- **Track 1(近期・構築能力)**: SNN がスパイクをデコード → Blender bpy 操作トークン列
  (プリミティブ追加/押し出し/細分化/アーマチュア追加/自動ウェイト/スマートUV展開/
  マテリアル割当)。リギング・UV・テクスチャを Blender 機能として得る。
- **Track 2(深部・フルジオメトリ)**: SNN が occupancy/SDF 場でジオメトリを、
  別ヘッドで UV・スキンウェイト・マテリアルを直接回帰。dense 出力と低電力主張の
  緊張は、スパイキングデコーダと最終段リファインメント限定で緩和。

### 5. Web 学習器バグ修正 `src/dst_snn/web_autonomous_learner.py`

- **novelty 常時 ≒ 1.0**: `encode()` が全トークンを `token_to_index` へ登録した後に
  「登録済みか」を判定しているため新奇性が機能しない。エンコード前の既知集合で判定するよう修正。
- **報酬が SNN 重みに効いていない**: 学習は活性トークンへの BCE(自己教師)で重みを
  更新し、報酬は関連メモリ側にのみ作用。報酬を SNN 学習信号へ結合する
  (報酬変調 or 損失重み付け)。目標指向学習(Phase 1)の頭金。

## リポジトリ統合

- 学習正典 = PyTorch DST-SNN (`src/dst_snn/`)。サロゲート勾配・自律巡回・研究モジュールを持つ。
- `benchmarks/` を新設(neuromorphic, threedcg)。
- EDEN/ブラウザ = 身体環境 + 可視化。Nim = 参照実装。役割を固定し断片化を止める。

## テスト方針

- `metrics.py` / 各スコアラ関数は既知入出力の単体テストで検証(合成メッシュ・
  既知スパイクテンソル)。
- スコアラは自明なベースライン生成器 + 手製参照メッシュで end-to-end 検証。
- ニューロモルフィックランナーは小サブセットでのスモークテスト(学習が進み精度が
  チャンスを上回ること)を CI 相当で確認。

## 環境上の注意

- 本環境に PyTorch は未インストール(`import torch` 失敗を確認)。実装は
  correct-by-construction を旨とし、torch/bpy 依存部は実データ検証を
  別途要する旨をコード/README に明記する。

## 未解決事項

- SketchFab 参照コーパスの具体的選定(ライセンス・カテゴリ・リグ有無の分布)は
  `benchmarks/threedcg/corpus.md` で別途詰める。
- 対LLM ベースラインで使う具体的 LLM/API は実装計画時に決定。
